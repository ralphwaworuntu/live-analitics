# uipro init for Cursor

## When to use

- The user wants **UI/UX Pro Max** (design-style databases, UX workflows) in **Cursor**.
- They mention **`uipro`**, **`uipro-cli`**, or **installing the Cursor workflow** without choosing another IDE.

## Prerequisites

- **Node.js** (v20+ recommended for tooling in this ecosystem).
- **`uipro` on PATH**, usually via:

  ```bash
  npm install -g uipro-cli
  ```

  If `uipro` is missing, **`npx`** works without a global install:

  ```bash
  npx uipro-cli init --ai cursor
  ```

## Command (project root)

From the **repository root** (where `.cursor` should live):

```bash
uipro init --ai cursor
```

This selects **Cursor** without an interactive assistant picker.

## Common flags

| Flag | Use |
|------|-----|
| `--ai cursor` | Install for Cursor only (no prompt). |
| `--force` | Overwrite an existing UI/UX Pro Max install. |
| `--offline` | Prefer bundled assets; use when network is restricted (see upstream docs for `--legacy` combinations). |

## What gets created (Cursor)

Current `uipro-cli` templates install the skill under **`.cursor/skills/ui-ux-pro-max/`** (see [upstream repo](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill)):

```
.cursor/skills/ui-ux-pro-max/
├── SKILL.md
├── data/
└── scripts/
```

**Python 3.x** is required for `scripts/search.py` (design system / search). On Windows, `python` or `python3` must run successfully.

Older docs may mention `.cursor/workflows/` plus `.shared/`; trust the files actually generated by your CLI version.

## After install

1. **Restart Cursor** so workflows/skills load.
2. Smoke-test with a short prompt (example from installer docs): *Build a landing page for a SaaS product*.

## Agent behavior

- Prefer **running** the command in the project root over pasting long boilerplate, unless the user only asked for documentation.
- If files already exist and the user wants a clean reinstall, suggest **`uipro init --ai cursor --force`** (confirm destructive overwrite when ambiguous).
- Do not confuse this with **Locofy UIPro** (`uipro.dev` MCP/Figma); that product uses separate MCP setup. This skill is for **`uipro-cli`** **UI/UX Pro Max** installer only.
